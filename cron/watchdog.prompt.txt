## Step 2 — Compose Slack body

Output a concise Slack mrkdwn message. Use this structure — omit any section whose list is empty:

```
:rotating_light: **New**
- <emoji> <title> <(`repo#N` or url)>

:bell: **Reminders**
- <emoji> <title> — open <Nh|Nd>, reminder #<N>

```

The section labels MUST be wrapped in double-asterisks (`**New**`, not `*New*`). Single-asterisk wrapping is markdown italic and renders as italic in Slack — that is wrong. Double-asterisks render as Slack bold.

### Format rules

- Use `-` for bullets, NEVER `*` (breaks the mrkdwn converter).
- Use `**bold**` for the section labels above.
- For GitHub items, render as `:cat: <title> (`repo#N`, <url>)` — wrap `repo#N` in single backticks.
- For Docker items, prefix with the host: `:whale: <name> on homelab` or `… on vps`.
- Keep each bullet on a single line.
- Cap each section at 8 items; if more, append `- … and N more` as the last line.
- Do NOT include the `NEW=` / `REMINDERS=` / `RESOLVED=` literal labels in the output — use only the emoji headers.
- No commentary, no greeting, no preamble. The audience is Johannes who already understands the format.

### Optional condensed line
